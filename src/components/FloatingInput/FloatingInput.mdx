## Examples

### Filled state

When the input has a value, the label stays floated above the input.

<CollapsibleCanvas of={FloatingInputStories.Filled} />
<CollapsibleSource of={FloatingInputStories.Filled} />

### With error message

Use the `errorMessage` prop to display a validation error below the input. When `errorMessage` is provided, the input automatically enters the invalid state.

<CollapsibleCanvas of={FloatingInputStories.WithError} />
<CollapsibleSource of={FloatingInputStories.WithError} />

### Clear button

Use the `onClear` prop to display a built-in clear button. The button appears when the input has a value and is not disabled or read-only.

<CollapsibleCanvas of={FloatingInputStories.WithClearButton} />
<div className="canvas-controls-only">
<Controls of={FloatingInputStories.WithClearButton}/>
</div>
<CollapsibleSource of={FloatingInputStories.WithClearButton} />

### Disabled

Use the `disabled` prop to disable the input, greying it out and preventing user interactions.

<CollapsibleCanvas of={FloatingInputStories.Disabled} />
<div className="canvas-controls-only">
<Controls of={FloatingInputStories.Disabled}/>
</div>
<CollapsibleSource of={FloatingInputStories.Disabled} />

### Read only

Use the `readOnly` prop to make the input read-only. The user can still select and copy the text.

<CollapsibleCanvas of={FloatingInputStories.ReadOnly} />
<CollapsibleSource of={FloatingInputStories.ReadOnly} />
